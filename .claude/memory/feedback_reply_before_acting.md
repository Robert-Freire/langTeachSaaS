---
name: Reply with understanding before acting on feedback
description: After receiving feedback (email, audio, call), always reply with summary + planned actions and wait 4 days before creating issues
type: feedback
---

When processing feedback from any source (Jordi, Teacher B, any teacher):

1. Transcribe/save raw feedback
2. Update the person's feedback log
3. **Reply to the person** with:
   - A summary of what you understood from their feedback
   - A list of issues you plan to create/update (high-level description, not yet created)
   - Ask them to confirm or correct if something was missed or misunderstood
4. **Set a reminder for 4 days later** to check for a reply
5. **Do NOT create GitHub issues yet**
6. After 4 days: if they replied, incorporate corrections first, then create issues. If no reply, proceed with creating issues as planned.

Reason: avoids acting on misunderstood feedback and gives the person a chance to correct or add context before we commit to specific issues.
