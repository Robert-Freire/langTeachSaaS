---
name: Report observations not acted upon
description: At the end of every summary, list things noticed but not fixed, so nothing silently drops
type: feedback
---

When wrapping up a piece of work or giving a summary, always include a "Noted but not acted upon" section listing anything observed during the work that was intentionally deferred, out of scope, or decided against. Each item should have a brief reason for deferral.

This prevents observations from silently disappearing. The user can then decide whether to act on them, create issues, or explicitly dismiss them.

Format:
```
**Noted but not acted upon:**
- <observation> — <reason deferred>
- <observation> — <reason deferred>
```

If there are no deferred observations, omit the section entirely (don't write "none").
