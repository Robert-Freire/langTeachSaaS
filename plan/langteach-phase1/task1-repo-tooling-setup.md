## Implementation Notes (Deviations from Original Plan)

1. **.NET 9 instead of .NET 8**: SDK 8 was not installed on the machine (only 9 and 10 available). All packages pinned to 9.x.
2. **Solution file is `.slnx`**: The `dotnet new sln` command now generates `LangTeach.slnx` (new XML format) instead of the legacy `.sln`.
3. **SQL Server on port 1434**: Port 1433 was already in use by an existing `resolver-sql-server` container. Mapped to host port 1434 instead.
4. **Full Docker Compose**: All three services (SQL Server, API, Frontend) are containerized, not just SQL Server. The frontend uses a volume mount for `src/` to enable hot reload during dev.
5. **No Swagger**: The API was scaffolded with `--no-openapi`. Swagger can be added in a later task if needed.
6. **SA_PASSWORD env var**: Uses `MSSQL_SA_PASSWORD` (current recommended name) with value `LangTeach_Dev1!`.

## 2. Docker Compose (as built)

Three services with health checks and dependency ordering:

```yaml
services:
  sqlserver:       # mcr.microsoft.com/mssql/server:2022-latest, port 1434:1433, healthcheck
  api:             # .NET 9 Web API, port 5000, depends_on sqlserver (healthy)
  frontend:        # Node 24 Alpine, Vite dev server, port 5173, volume mount src/
```

Connection string inside Docker uses `Server=sqlserver,1433` (container-to-container).
Connection string for local dev (outside Docker) uses `Server=localhost,1434`.
