# Task 127 — Show Correct Answer + AI Explanation on Wrong Exercise Response

## Summary
Add an `explanation` field to all exercise types (pre-generated by AI at creation time), display the correct answer and explanation in the student view when an answer is wrong, and show it as a read-only preview in the editor.

---

## Approach: Pre-generation

Explanation is generated alongside the exercise content at lesson creation time (stored in `GeneratedContent` JSON). No new endpoint or on-demand AI call needed. Simpler, no latency during student interaction.

---

## Backend

### 1. Update `PromptService.BuildExercisesPrompt`
File: `backend/LangTeach.Api/AI/PromptService.cs`

Add `explanation` to the JSON schema in the system prompt:
```json
{
  "fillInBlank": [{"sentence":"","answer":"","hint":"","explanation":""}],
  "multipleChoice": [{"question":"","options":[""],"answer":"","explanation":""}],
  "matching": [{"left":"","right":"","explanation":""}]
}
```

Update the user prompt to instruct: "For each exercise, include a concise explanation (2-3 sentences) of why the correct answer is correct, considering the student's level and common L1 interference patterns."

No model, migration, or controller changes needed.

---

## Frontend

### 2. Update types (`frontend/src/types/contentTypes.ts`)
Add `explanation?: string` to:
- `ExercisesFillInBlank`
- `ExercisesMultipleChoice`
- `ExercisesMatching`

### 3. Update `ExercisesRenderer.tsx` — Student view

When a student answers incorrectly and checks answers, show below the wrong answer:
- Correct answer (e.g. "Correct answer: **fue**")
- Explanation text (if present): shown in a subtle info box

Location in file: Student export, inside the feedback rendering for each exercise type (fill-in-blank ~line 400, multiple choice ~line 490, matching ~line 570).

### 4. Update `ExercisesRenderer.tsx` — Editor view

In the Editor export, add a read-only "Explanation" row/field per exercise so the teacher can preview the AI-generated explanation. Use a subtle gray text or tooltip-style display (not an input — teachers edit via regeneration).

## Design decisions

- **Pre-generation only**: No on-demand generation. Exercises without explanation (legacy content) degrade gracefully — show correct answer only.
- **Editor view**: Read-only display (not editable). Teachers regenerate if they want a different explanation.
- **Matching explanations**: Optional (matching exercises are less common and explanations are less obvious). Include if AI generates them, skip gracefully if absent.
