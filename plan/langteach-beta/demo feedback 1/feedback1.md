1. CEFR Curriculum Alignment
The product should integrate the Common European Framework of Reference for Languages (CEFR) levels (A1–C2) to define learning objectives and ensure appropriate content progression at each stage.

No solo, también podría ser útil para cursos de conversación o de cursos específicos (negocios, Derecho...) 
2. Placement Testing
Include functionality to generate diagnostic assessments that accurately determine a student’s initial proficiency level.

3. Lesson Plan Generation
Enable automated creation of structured learning plans based on:

Target level (e.g., B2)

Total number of sessions

Instructional hours

Plans should systematically cover both grammar and communicative competencies.

Que cubra 4 competencias ( comprensión escrita, Expresión escrita, comprensión auditiva, expresión oral) y actividad más orientadas a la parte gramatical con teoría y práctica. Que también pueda ayudar al profesor a explicar conceptos más complicados...
4. Per-Session Activity Planning
For each lesson, generate specific, actionable activities rather than generic suggestions, ensuring pedagogical relevance and variety.

5. Student Profile Personalization
Incorporate learner-specific variables, such as:

Native language (L1)

Group composition

Use this data to address language interference patterns and tailor instruction to student needs.

Que permita incluir dificultades específicas de diferentes alumnos o del grupo en general ( por ej, a un alumno le cuesta la diferencia entre ser, estar hay.y a otra la pronunciación....)
6. Support for In-Person Instruction
Design the system to accommodate both:

Online teaching environments

Face-to-face (presential) classrooms

7. Class Tracking and Adaptive Follow-Up
Allow instructors to record post-class audio reflections, which the system analyzes to:

Update student progress

Adapt subsequent lesson plans dynamically

8. Evaluation Tools
Provide advanced text correction and feedback mechanisms that:

Classify errors by type (e.g., grammar, vocabulary, punctuation, verb forms)

Deliver more granular analysis than generic AI outputs

9. Custom Material Upload
Teachers should be able to upload proprietary teaching resources (e.g., PDFs, worksheets, slides) to enrich the system’s content base and ensure alignment with their existing methodology.

10. Presentation and Slide Generation
The system should support automated creation of instructional assets, including:

Slide decks

Visual aids

Classroom-ready materials

These outputs should be pedagogically structured and adaptable to lesson objectives.

El material debería ser variado, desde posible infografías didácticas y explicativas a materiales diversos para practicar, crucigramas, juegos como el de la oca, tableros de errores, vídeos interactivos, etc...
11. Material-Assisted Planning
Enable workflows where uploaded materials + student profile data are jointly leveraged to:

Generate curriculum plans

Design session-specific activities

Example use case: given a specific resource and a defined learner group, produce a targeted lesson plan for a single session.

Y que al acabar la sesión se pueda readaptar las siguientes clases, en función de las dificultades o facilidades encontradas ( esto no debe ser una carga administrativa para el profesor, por eso se me ocurre lo del audio)
